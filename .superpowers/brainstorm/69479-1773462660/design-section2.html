Full Design — Section 2: Role State Machine & Dispatch Logic

How the coordinator decides when to send work to an agent

Role State Machine (per role)
free
No task running
Queue may have items
SDK not in flight
working
sdk.query() in flight
Task status: in_progress
Do not dispatch
waiting_human
SDK returned BUT
task still in_progress
⚠️ Alert dashboard
ready
SDK returned AND
task done/rejected
Dispatch next now
State Transitions
free  + task in queue                     → pop task, call sdk.query() → working
working  + sdk.query() returns + task done/rejected  → ready
working  + sdk.query() returns + task in_progress   → waiting_human
ready    + task in queue                     → pop task, call sdk.query() → working
ready    + queue empty                      → free
waiting_human + task file updated to done/rejected  → ready → dispatch next
waiting_human + human sends message to agent      → call sdk.query() again → working
Dispatch Logic (coordinator code sketch)
// called on: fs.watch event OR sdk.query() completes
async function tryDispatch(role) {
  const runner = roles[role]
  if (runner.sdkInFlight) return  // working — do nothing

  const currentTask = runner.currentTask
  if (currentTask && currentTask.status === 'in_progress') {
    runner.state = 'waiting_human'  // alert dashboard
    return
  }

  const next = runner.queue.shift()
  if (!next) { runner.state = 'free'; return }

  runner.state = 'working'
  runner.sdkInFlight = true
  runner.currentTask = next
  const result = await sdk.query(buildPrompt(next), { session_id: role })
  runner.sdkInFlight = false
  runner.currentTask = rereadTaskFile(next)  // check real status
  tryDispatch(role)  // recurse — check if we can dispatch next
}
Queue Rules
• Tasks enter queue sorted by priority (high → medium → low) then created date
• A role's queue only holds tasks with status: pending and to: <this role>
• If a task is rejected, notify the from role's queue (same dispatch logic applies)
• Queue is rebuilt on server start by scanning all tasks/*.md files
